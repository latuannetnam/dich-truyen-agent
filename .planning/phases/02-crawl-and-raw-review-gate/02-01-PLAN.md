<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| URL/profile input -> outbound HTTP | User URLs and YAML profile domains must not authorize arbitrary cross-domain crawling. |
| External bytes -> decoded HTML | Encoding declarations and bytes are untrusted and may be malformed. |
| External HTML -> catalog/raw text | Links, titles, scripts, and residue must be validated before persistence. |
| Local override -> shared template | A book-local repair must not mutate reusable domain rules without explicit promotion. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Tampering / SSRF | `crawl_profiles.py` URL and domain resolution | mitigate | Require expected HTTP(S) host equality for shared lookup, local override validation, pagination traversal, and promotion; test wrong-domain rejection. |
| T-02-02 | Tampering | `crawler.py` decoding | mitigate | Use bounded precedence and fallback encodings; reject unusable text rather than silently ignoring decode failures. |
| T-02-03 | Tampering | `crawler.py` catalog discovery | mitigate | Block duplicate links/source IDs and clear parsed ordinal gaps; never silently dedupe or trust URL IDs as ordinals. |
| T-02-SC | Tampering | dependency install | mitigate | Install only packages approved in `02-RESEARCH.md`; lock exact resolutions with `uv.lock`. |
</threat_model>
