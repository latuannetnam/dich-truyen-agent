<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Report/raw files -> approval checkpoint | Reviewed files can mutate after approval and must invalidate downstream access. |
| Partial crawl -> downstream full-book workflow | A deliberately limited test crawl must never masquerade as full approval. |
| Agent repair -> shared domain template | A local repair may be wrong for other books and requires explicit promotion. |
| CLI/skill output -> agent context | Reports and bodies remain on disk; terminal output stays compact. |

| T-02-09 | Tampering | partial approval scope | mitigate | Persist `ApprovalScope`; add full-scope gate helper and tests proving partial rejection. |
| T-02-10 | Tampering | profile promotion | mitigate | Keep repair workspace-local until separate explicit promotion command validates matching domain. |
