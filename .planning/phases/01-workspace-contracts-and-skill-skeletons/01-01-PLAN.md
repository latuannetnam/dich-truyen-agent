<planning_notes>
The hardest constraint is interruption-safe persistence: if atomic replacement or strict resume
validation is wrong, every downstream stage can silently corrupt or overwrite book work. Task 2
implements that constraint before Task 3 exposes workspace initialization.

Pre-mortem checks:
- A slug or persisted relative path escapes `books/`; catch it with traversal tests before any
  filesystem mutation.
- A direct canonical write bypasses interruption safety; keep canonical writes behind
  `atomic_write_yaml()` and assert orphan siblings never replace canonical data.
- Resume trusts state markers without checking files; assert missing and hash-mismatched completed
  artifacts return blocking compact results.
</planning_notes>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| CLI/service input -> filesystem paths | Book slugs, canonical relative paths, and configured roots are untrusted until resolved beneath the expected books root. |
| YAML file -> Pydantic model | Human-edited or interrupted persisted files can be malformed or malicious. |
| Temporary sibling -> canonical file | A partial or invalid write must never replace the last valid canonical file. |
| Mutable state -> completed artifact | A completed marker is not trusted until the referenced file exists and its SHA-256 matches. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | Tampering | `paths.py` slug and relative-path handling | mitigate | Reject separators, `..`, absolute paths, empty slugs, and resolved paths outside the configured workspace root before mutation; cover with traversal tests. |
| T-01-02 | Tampering | `storage.py` YAML loader | mitigate | Parse only with `yaml.safe_load()` and validate through explicit Pydantic 2 models; test unsafe tags and malformed YAML. |
| T-01-03 | Tampering | `workspace.py` initialization | mitigate | Refuse an existing workspace unless explicit resume is selected per D-05; assert existing content remains byte-for-byte unchanged. |
| T-01-04 | Tampering / Denial of Service | `storage.py` canonical promotion | mitigate | Write sibling temp, flush, fsync, reload and validate, then `os.replace()`; report but never auto-promote or auto-delete orphan temps per D-07. |
| T-01-05 | Tampering | `workspace.py` resume | mitigate | Recompute completed artifact hashes and stop on missing, malformed, or mismatched canonical files per D-06 and D-08. |
| T-01-SC | Tampering | Python dependency installs | mitigate | Install only packages marked Approved in `01-RESEARCH.md` Package Legitimacy Audit; do not add unaudited direct dependencies. |
</threat_model>
