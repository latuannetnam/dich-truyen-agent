<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Reviewed workspace files -> approval record | Evidence can change after review and must invalidate approval. |
| Custom or bundled YAML -> style snapshot | Style YAML is human-editable input and must never construct arbitrary objects. |
| CLI arguments -> workspace services | Paths and checkpoint types remain untrusted until validated by shared helpers. |
| Helper result -> agent context | Output must remain compact and avoid leaking chapter bodies or verbose logs. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-06 | Tampering | `checkpoints.py` stale approvals | mitigate | Recalculate every evidence SHA-256 at each gate check; block on mutation, deletion, or malformed approval per D-10. |
| T-01-07 | Tampering / Elevation of Privilege | `styles.py` and checkpoint YAML loading | mitigate | Use shared `yaml.safe_load()` plus explicit Pydantic models; test unsafe tags, malformed YAML, and unknown checkpoint types. |
| T-01-08 | Tampering | `checkpoints.py` report and evidence paths | mitigate | Require workspace-relative paths and reject traversal or resolved paths outside the selected workspace. |
| T-01-09 | Tampering | `styles.py` workspace snapshot | mitigate | Validate selected style then write workspace-local `style.yaml` through atomic storage; resume uses the snapshot per D-13. |
| T-01-10 | Information Disclosure | `cli.py` and skill contracts | mitigate | Restrict output to `OperationResult` status, reason, progress, and report or approval paths; test absence of chapter-body content. |
| T-01-11 | Tampering | `workspace.py` init called through CLI | mitigate | Preserve Plan `01-01` overwrite refusal: existing book workspaces block unless explicit `--resume` is supplied. |
| T-01-SC | Tampering | Python dependency installs | accept | This plan adds no dependency install; it consumes only Plan `01-01` packages approved by the research legitimacy audit. |
</threat_model>
